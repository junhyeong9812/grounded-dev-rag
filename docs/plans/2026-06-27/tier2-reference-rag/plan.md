# Tier 2 계획서 — 레퍼런스 RAG (짓기 전 계획)

> 상태: **탐색/계획** (아직 빌드 아님 — off-ramp 열려 있음)
> 상위 맥락: local-llm 구상 → "방법론은 harness 강화(Tier 1, 완료·푸시), 레퍼런스는 별도(Tier 2)" 결정. 이 문서는 Tier 2를 짓기 전 계획.

## 0. Tier 2가 무엇인가 (한 문장)

DDD 저자·Spring·고수들의 **"왜 좋은가"를 설명한 자료를 큐레이션·임베딩**해서, 코딩 시 Claude Code가 MCP로 관련 레퍼런스를 끌어오게 한다.

## 1. 가장 중요한 결정 — 콘텐츠 우선

서버(인프라)는 30분이면 띄우지만, 넣을 좋은 자료가 없으면 빈 검색엔진이다. 그래서:

```
콘텐츠 소량(20~30개) 먼저 → 검색이 실제로 가치 내는지 확인 → 그때 인프라 풀가동
```

## 2. 콘텐츠 전략 (진짜 일)

| 항목 | 결정 |
|------|------|
| 무엇 | 패턴을 *설명한 문서* 1순위(임베딩 잘 됨) · 코드는 그 문서에 링크/첨부(날코드 임베딩은 노이즈) |
| 출처 | DDD(Evans/Vernon)·Fowler·Spring 설계 해설·고수 블로그 → **내 언어로 압축**(품질↑·라이선스 클린). 방법론 원본 4레포는 `/home/jun/project/레퍼런스/`에 보관(이동 완료) |
| 단위 | 1 자료 = 1 패턴/원칙 (예: "Aggregate 경계 잡는 법", "재시도 idempotency") |
| 라벨 | 도메인·패턴유형·언어·출처 메타 부착 (필터·클러스터링용) |
| 형식 | 마크다운 — 제목/원칙/왜/코드예시(링크)/안티패턴 |
| 큐레이션 콘텐츠 위치 | 미정 — 후보: `레퍼런스/curated/` 또는 `local-llm/corpus/` |

```
1. .164 GPU 기반   — NVIDIA 드라이버+CUDA+Docker (지금 드라이버 0)  ※7B 쓸 때만

```

## 5. 중단 기준 (off-ramp)

- harness 방법론 렌즈만으로 Claude 출력이 충분히 좋아지면 → **Tier 2 보류**(인프라 안 지음).
- 콘텐츠 20개 만들었는데 "검색해 끌어올 양이 안 된다" 느끼면 → MCP 없이 **그냥 파일로** harness에 흡수.

## 6. 남은 미결정

1. **벡터DB**: Qdrant(가볍고 하이브리드 전용) vs **ES**(사용자 ES 경험·nori 한국어 recall↑) — ES가 유리할 수 있음.
2. **순서**: 콘텐츠 먼저(추천) vs 인프라 먼저.
3. **큐레이션 콘텐츠 저장 위치** (2번 표 참조).

## 7. 첫걸음 (제안)

콘텐츠 1~2개 시범 제작 — 예: "Aggregate 경계 잡는 법"을 Evans/Vernon 기반 한 장. 이게 *실제로 Claude에 줄 레퍼런스가 되는지* 감을 잡고 → 포맷 확정 → 양 확대.
